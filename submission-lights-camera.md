### Design Choices
The following discusses **2.4 Data Handling** which mostly covers the GLSL portion. The rest lighting/parsing algorithm is similar to the previous projects. 

The main work lies inside the ```realtime.cpp``` file composing of three sections, ```initializeGL()```,  ```paintGL()```, ```settingsChanged()```, and ```sceneChanged()```. Lets start from talking about ```sceneChanged()``` since it is the easiest to explain one.

The ```sceneChanged()``` function involves loading a new scene. Whenever a json file is read, we need to parse and store objects, lights, and other parameters into our program and render them later. This can be achieved by the ```SceneParser::parse``` method. 

For ```initializeGL()```, we need to setup the VBO's and VAO's for our primitives. In my program, I use *four pairs of VBO's and VAO's* with respect to each of the primitive we want to render. Given parameter 1 and parameter 2 during our tessellation, the number of triangles is fixed for each primitive. As a result, we can use 4 different VBO's to setup a fixed buffer size that stores the vertex and normal information. The primitive class is cloned into the codebase from Lab8 with tessellation methods implemented for cone and cylinder. See ```setVboAndBindVao(GLuint, GLuint, const vector<float>&)``` for details on the sequence for setting up and binding VBO's with VAO's. 

The ```paintGL()``` is essential for *sending parameters to the shader program*. I am lazy explaining each uniform that is being sent, so I will just list out all the uniforms, namely, "ViewMatrix", "ProjectionMatrix", "campos", "ka", "kd", "ks", "num_lights", "lights_array.color", "lights_array.pos", "lights_array.dir", "lights_array.function", "lights_array.type", "lights_array.penumbra", "lights_array.angle", "ModelMatrix", "cAmbient", "cDiffuse", "cSpecular", "cReflective" (not used though), "shininess". These uniform are useful for setting up the transformation inside the vertex shader, as well as the lighting algorithm inside the fragment shader. See the ```Camera``` class to check for details on how I *spelled out* the perspective matrix. With all uniform being passed, we have all the critical information to render each object stored in the *m_renderData* variable. To draw, we need to call ```glBindVertexArray(m_shape_vao)``` depending on the primitive of interest, then call ```glDrawArrays(GL_TRIANGLES, 0, num_triangles_to_draw)```. And each iteration we call bind a different VAO, so at the end of each iteration, we unbind the VAO by ```glBindVertexArray(0)```.

As much as we have discussed how the VAO's are dependent to the primitive and parameters 1/2, whenever we change the settings, especially when parameter 1 and 2 are modified, we need to reinitialize the VBO's and VAO's since the number of vertices no longer apply. We take care of passing the new vertex + normal vector to each primitive's VBO then bind it with their VAO respectively inside the ```settingsChanged()``` function.
